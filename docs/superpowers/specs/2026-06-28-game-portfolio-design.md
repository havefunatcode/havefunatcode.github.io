# 게임형 포트폴리오 랜딩 — 설계 문서 (Design Spec)

- 작성일: 2026-06-28
- 대상 사이트: https://havefunatcode.github.io
- 레포: `havefunatcode/havefunatcode.github.io`
- 작업 브랜치: `feature/game-portfolio`

## 1. 개요 / 목표

방문자가 **횡스크롤 플랫포머(마리오풍)** 미니게임을 플레이하여, 캐릭터를 조작해
스테이지 끝의 **보물상자**에 도달하면 **이태호의 포트폴리오(PDF)**가 열리는 인터랙티브
랜딩 페이지를 만든다.

- 조작: `←/→` 이동, `Space`/`↑` 점프, 보물상자 앞 `Space` 열기
- 보물 = `이태호_포트폴리오.pdf` (게임 내 모달로 노출)
- 채용담당자/모바일을 위한 **건너뛰기(skip)** 경로 보장
- 기존 Gatsby 블로그는 게임으로 **대체**하되 원본은 안전하게 보존

### 성공 기준 (한 줄)

"게임을 즐기든 건너뛰든, 모든 방문자가 막힘없이 포트폴리오 PDF에 도달한다."

## 2. 사용자 흐름

```
[타이틀]                  [플랫포머 1스테이지]                 [보물 모달]
"이태호 | 서버 개발자"  →  @ 우측 이동, 구덩이/발판 점프    →  🎁 도착 + Space
"▶ 시작하기"               경로에 프로젝트 코인 7개             → 포트폴리오 PDF 뷰어
"⏭ 이력서 바로 보기"        끝에 보물상자 🎁                     + GitHub + 이메일
   │
   └─(skip)──────────────────────────────────────────────→ [보물 모달] 직행
```

- **첫 진입**: 타이틀 화면. `시작하기` 또는 `⏭ 이력서 바로 보기`.
- **플레이**: 가로 스크롤 스테이지. 약 20~40초 분량. 적/전투 없음(가벼운 점프 위주).
- **도달**: 보물상자에 닿아 `Space` → 상자 열림 연출 → 보물 모달.
- **스킵**: 언제든 우측 상단 `⏭` 버튼으로 모달 직행.

## 3. 게임 디자인 명세

### 3.1 조작 (입력)
- 데스크톱: `←/→`(또는 `A/D`) 좌우, `Space`/`↑`/`W` 점프, 보물상자 근접 시 `Space` 상호작용.
- 모바일/터치: 좌하단 반투명 `◀ ▶`, 우하단 `⤒(점프)`. 보물상자 근접 시 점프 버튼이 `열기`로 전환.
- 입력은 `keydown/keyup` + 터치 이벤트를 공통 `input` 상태 객체로 정규화.

### 3.2 물리 (engine)
- 2D AABB 충돌. 중력 가속도 적용, 점프는 순간 상향 속도.
- 좌우 이동 속도, 점프 높이는 상수로 튜닝 가능하게 분리.
- 바닥/발판 위에서만 점프 가능(코요테 타임 ~80ms 허용으로 손맛 개선 — 선택).

### 3.3 레벨 (level)
- **단일 스테이지**. 데이터는 좌표 배열로 정의(발판, 구덩이, 코인, 상자, 시작점).
- 난이도: 비게이머도 클리어 가능한 수준. 구덩이 1~3칸 점프, 발판 1~2단 오르기.
- **프로젝트 코인 7개** = 실제 프로젝트 7선. 코인 획득 시 상단 토스트에 프로젝트명 표시.
  수집은 보너스이며 진행 필수 아님.
- **리스폰**: 구덩이에 빠지면 처음이 아니라 **마지막으로 밟은 안전 발판**에서 부활(좌절 최소화).

### 3.4 카메라
- 캐릭터를 가로로 추적하는 사이드 스크롤 카메라. 세로는 고정 또는 약한 추적.

### 3.5 연출
- 코인 획득 SFX + 작은 파티클.
- 보물상자 도달 → 상자 열림 스프라이트 → 빛/파티클 → 0.4s 후 모달 페이드인.
- `prefers-reduced-motion` 시 파티클/흔들림 최소화.

## 4. 보물 모달 명세 (= 포트폴리오)

게임 캔버스 위에 오버레이되는 모달. 구성:

- **헤더**: "이태호 · 4년차 서버 개발자" + 한 줄 소개
  ("대용량 데이터 처리 / 시스템 아키텍처 개선 / 서비스 안정화")
- **하이라이트 스탯(3~4개 칩)**: 예) "검색 서버 재기동 0회", "응답 속도 75%↓",
  "공유링크 50만건 · OKR S등급" — 시선을 끄는 요약(상세는 PDF).
- **포트폴리오 뷰어**:
  - 데스크톱: 모달 내부에 PDF 인라인 뷰어(`<object>`/`<iframe>`)로 `portfolio.pdf` 임베드.
  - 공통 액션 버튼: `📄 새 탭에서 열기`, `⬇ PDF 다운로드`.
  - 모바일: 인라인 PDF 렌더가 불안정하므로 큰 버튼(새 탭/다운로드)만 노출.
- **연락/링크**: `GitHub (havefunatcode)`, `이메일 (dlxogh561@gmail.com)`.
- **닫기**: `Esc` 또는 X. 닫으면 게임으로 복귀(또는 타이틀).

> 보물 PDF는 `00_sources/career/이태호_포트폴리오.pdf`(약 1.1MB, 8p)를 게임 레포로
> 복사하여 **사이트 루트의 `portfolio.pdf`**로 호스팅한다(ASCII 파일명으로 URL 단순화).
> 원본 위키 소스는 읽기 전용이며 수정하지 않는다(복사만).

## 5. 기술 아키텍처

**A안 — 순수 HTML5 Canvas + Vanilla JS (빌드 없음, 의존성 0).**

```
index.html            엔트리, <canvas>, 타이틀/HUD/모달 마크업, 스크립트 로드
css/style.css         레이아웃, image-rendering:pixelated, 모달/터치패드 스타일
js/main.js            부트스트랩: 에셋 로드 → 상태기계(타이틀/플레이/모달) 시작
js/engine.js          게임 루프(rAF), 시간 델타, 입력 정규화, AABB·중력, 카메라
js/level.js           스테이지 데이터(발판/구덩이/코인/상자/시작점) + 충돌 질의
js/sprites.js         이미지/스프라이트시트 로딩 및 그리기 헬퍼
js/audio.js           SFX 로드/재생, 음소거 토글(localStorage)
js/ui.js              타이틀, HUD, 토스트, 보물 모달, 스킵 버튼, 모바일 터치 컨트롤
data/portfolio.js     콘텐츠 데이터(프로필/하이라이트/프로젝트/링크) — 코드와 분리
assets/               캐릭터·타일·코인·상자 스프라이트, SFX
portfolio.pdf         보물 PDF (사이트 루트)
```

### 모듈 책임 / 인터페이스 (요지)
- `engine.js`: `update(dt)`, `render(ctx)`, 충돌은 `level.solidAt(x,y,w,h)`에 위임.
  외부엔 게임 상태 전환 콜백(`onReachChest`)만 노출.
- `level.js`: 순수 데이터 + 질의 함수. 렌더/입력 모름.
- `ui.js`: DOM만 다룸. `openTreasureModal(data)` / `showSkip()` / `mountTouchControls(input)`.
  게임 로직 모름. 게임→UI는 콜백/이벤트로만 통신.
- `data/portfolio.js`: 표시용 데이터. 경력 변경 시 이 파일만 수정.

각 모듈은 단일 책임을 가지며 독립적으로 이해/교체 가능해야 한다.

## 7. 모바일 & 접근성
- 작은 화면/터치 디바이스 자동 감지 → 터치 D-패드 + 점프 버튼 오버레이.
- 키보드만으로 완주 가능. 모달 포커스 트랩, `Esc` 닫기.
- `prefers-reduced-motion` 존중. 사운드 기본 음소거 + 토글(localStorage 기억).
- 스킵 버튼은 타이틀/플레이 어디서나 노출 → 접근성 보장.

## 8. 에셋 & 사운드
- 픽셀 스프라이트: **CC0 라이선스 무료 에셋팩**(예: Kenney.nl 픽셀 플랫포머) 또는 미니멀 자체 제작.
  라이선스가 명확히 안전한 것만 사용하고 출처를 `assets/CREDITS.md`에 기록.
- SFX(점프/코인/상자): CC0. 기본 음소거, 토글 제공.

## 9. 배포 전략
현재: GitHub Pages가 `deploy` 브랜치 루트를 서빙(legacy). `main` = Gatsby 소스, `deploy` = 빌드본.

계획:
1. **보존**: 기존 Gatsby 사이트를 `archive/gatsby-blog` 브랜치 + 태그(`v-gatsby-blog`)로 백업(삭제 금지).
2. **구성**: `main`에 정적 게임 사이트(섹션 5 구조)를 구성. 빌드 불필요.
3. **전환**: GitHub Pages 소스를 **`main` 루트**로 변경(정적 직접 서빙).
   - 대안: `deploy` 브랜치 방식 유지를 원하면 정적 파일을 `deploy`로 복사·푸시.
4. 커스텀 도메인/`CNAME` 없음 확인(현재 `havefunatcode.github.io` 기본 도메인).
5. 작업은 `feature/game-portfolio` 브랜치에서 → 검토 후 `main` 반영.

## 10. 범위 / 비범위 (YAGNI)
- **포함**: 타이틀 → 1스테이지 플랫포머 → 보물 모달(PDF), 코인 7개, 스킵, 모바일 터치,
  PDF 호스팅, 보존 배포, 음소거 토글.
- **제외**: 적/전투, 멀티 스테이지, 점수/랭킹, 저장/로그인, 백엔드, 빌드 파이프라인,
  사운드 작곡, 다국어.

## 11. 리스크 & 완화
- 모바일 인라인 PDF 미지원 → 모바일은 버튼(새 탭/다운로드)로 대체.
- 에셋 라이선스 → CC0만 사용 + CREDITS 기록.
- 게임이 콘텐츠 도달을 막을 위험 → 스킵 버튼 + 쉬운 난이도 + 리스폰으로 완화.
- Pages 소스 전환 시 일시적 표시 변화 → 사전 백업(archive) 후 전환.
- PDF 1.1MB 초기 로딩 → 모달 열 때 지연 로드(lazy), 게임 초기엔 미로드.

## 12. 수용 기준 (Acceptance Criteria)
1. 데스크톱에서 방향키+스페이스로 캐릭터가 이동·점프하고 보물상자에 도달할 수 있다.
2. 보물상자 도달 + Space 시 모달이 열리고 `portfolio.pdf`를 보거나 다운로드할 수 있다.
3. 우측 상단 스킵 버튼으로 어느 시점에서든 모달에 즉시 도달한다.
4. 모바일(터치)에서 터치 컨트롤로 플레이 가능하고, PDF는 새 탭/다운로드로 열린다.
5. 기존 Gatsby 사이트가 `archive/gatsby-blog`로 보존되어 있다.
6. 페이지가 GitHub Pages에서 빌드 없이 정적으로 서빙된다.
